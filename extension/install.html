第一次使用要准备什么

下载与扩展版本一致的 macOS Engine ZIP，解压后双击“Install LocalTube Dub Engine.command”。安装程序会自动准备字幕读取、本地系统配音、可选自然在线配音和开机启动；日常使用不需要打开终端。有字幕视频完成这一步即可，无字幕视频再按需安装下面的本地转写组件。

当前私测包尚未签名和公证。如果 macOS 阻止首次打开，请右键安装文件并选择“打开”。

需要 Python 3.10+、yt-dlp 和正在运行的 LocalTube Dub Engine。macOS 一键依赖脚本会检查 Python 版本；如果只有苹果自带的 Python 3.9 且已经安装 Homebrew，会自动安装新版 Python，并为本项目创建独立环境。首次使用默认选择无需 API Key 的 Microsoft 自然在线语音，只发送待朗读字幕；需要完全本地配音时可改选系统自带 say。只看有字幕视频时不需要 Whisper；没有字幕的视频可一键安装本地 whisper.cpp 免费转写，也可以改用 Groq / Deepgram / OpenAI 转写 Key。

启动本地 Engine

完成安装后，Engine 会在登录 macOS 时自动运行。下面的按钮用于立即启动、重启或修复开机启动；正常使用通常不需要重复安装。

如果刚安装后仍显示未连接，请完全退出并重新打开 Chrome，再点“一键启动 Engine”。

Engine 会监听 127.0.0.1:8787。插件会优先使用页面已有目标语言字幕；页面读取失败或只有原文字幕时，再用 Engine 调用 yt-dlp 提取字幕文本。同步配音会先调用本地 TTS 生成音频片段并按视频进度播放，失败时再退回浏览器朗读。

首次 Native Host 安装会同时注册 macOS 登录自启动。以后重启电脑，系统会自动启动并守护 Engine；“修复开机自启”可以重新生成配置。如果终端看到 Address already in use，新版启动脚本和“一键重启 Engine”会自动清理确认属于 LocalTube Dub 的旧进程。如果刚安装 Native Host 后仍提示不可用，请完全退出并重启 Chrome。

升级开发版后要刷新

如果改了插件代码，先在 chrome://extensions 点 LocalTube Dub 的“重新加载”，再刷新已经打开的 YouTube 视频页。视频页右侧面板标题里的版本号必须和当前 manifest 版本一致，否则页面还在运行旧内容脚本。

429 限流处理

YouTube 返回 429 表示当前网络或 cookies 被临时限流。先等待几分钟，不要反复点击开始。Engine 默认读取 Chrome cookies；如果读取 cookies 失败，可以关闭 Chrome 后重启 Engine，或临时禁用 cookies。

正式商店发布准备

当前构建已经能生成可上传 Chrome Web Store 的扩展 ZIP，以及绑定指定扩展 ID、可双击安装和卸载的 macOS Engine 私测包。正式公开下载前仍需要 Apple Developer ID 签名与公证、固定下载地址、隐私政策和支持页面；Windows 安装器仍在后续计划中。
