# Security Policy

## Supported version

Security fixes are applied to the latest source revision and the latest published extension version. Older development ZIP files are not maintained.

## Reporting a vulnerability

Use GitHub private vulnerability reporting from the repository's **Security** tab. If private reporting is not yet enabled, open a minimal issue asking a maintainer to establish a private contact channel; do not include exploit details, API keys, cookies, personal data, or affected user information in a public issue.

Include:

- the affected version and platform;
- whether the issue affects the Chrome extension, local Engine, or installer;
- reproduction steps with secrets removed;
- the expected impact; and
- any suggested mitigation.

Please do not test against other users, third-party accounts, or systems you do not own or have permission to assess.

## Sensitive data

LocalTube Dub must never commit or log API keys, YouTube cookies, complete captured audio, or private caption content. A report containing sensitive data should use the private channel above.
